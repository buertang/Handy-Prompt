# 修复标签删除时重复生成“默认标签”的问题

## 问题分析
在 `TagManager.tsx` 中删除标签并选择“移动到默认标签”时，代码目前仅通过当前语言环境下的默认标签名称（如 "默认" 或 "Default"）来查找默认标签。
如果因为语言环境差异导致名称不匹配，代码会错误地认为默认标签不存在，从而创建一个新的同名标签。
更严重的是，新创建的标签缺少 `isDefault: true` 属性，导致它只是一个普通的同名标签，而非系统默认标签。

## 修复计划
修改 `entrypoints/options/pages/TagManager.tsx` 中的 `confirmDelete` 函数，优化默认标签的查找和创建逻辑。

### 具体步骤
1.  **优先查找系统默认标签**：
    - 使用 `db.tags.filter(t => !!t.isDefault).first()` 优先查找具有 `isDefault: true` 属性的标签。
    - 这种方式不受标签名称和语言环境的影响。

2.  **回退查找机制**：
    - 如果未找到系统默认标签，再尝试通过名称（`t('tag.defaultTag')`）查找现有标签。

3.  **正确创建默认标签**：
    - 如果上述两种方式都未找到标签，则创建一个新的默认标签。
    - **关键修复**：在创建新标签时，显式设置 `isDefault: true`，确保其被系统正确识别为默认标签。

### 代码变更点
- 文件：`entrypoints/options/pages/TagManager.tsx`
- 函数：`confirmDelete`
- 逻辑：
  ```typescript
  // 伪代码
  let defaultTag = await db.tags.filter(t => !!t.isDefault).first();
  const defaultTagName = defaultTag ? defaultTag.name : t('tag.defaultTag');

  if (!defaultTag) {
    // 尝试按名字查找
    defaultTag = await db.tags.where('name').equals(defaultTagName).first();
  }

  if (!defaultTag) {
    // 创建新的默认标签
    await db.tags.add({
      // ...
      name: defaultTagName,
      isDefault: true, // <--- 加上这个
      // ...
    });
  }
  ```
